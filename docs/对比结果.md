## 二、关键发现与分析

### 2.1 各 Phase 性能提升路径

```text
PPL (Phase 2)                  52.93%  ─── 概率基线
    ↓ +21.2%
SAPLMA LR L27 (Phase 2)        74.17%  ─── hidden state 明显优于序列概率
    ↓ +5.7% (层选择)
SAPLMA LR L17 (Phase 3)        79.87%  ─── 中后层优于最后层
    ↓ +0.95% (Linux + bfloat16 复跑)
Hidden-only FULL (Phase 4)     80.82%  ─── 新环境下全量基线更高
    ↓ +5.84% (子集效应)
Hidden-only subset (A0s)       86.67%  ─── 600/150/150 子集更容易
    ↓ +2.00% (top-head attention 融合)

```

补充：A8 的 Test AUROC = 0.9403，是当前复跑中最强的排序能力结果。

### 2.2 Attention 特征分析

| 发现 | 详情 |
|------|------|
| **Linux 环境本身不够** | 在 Linux + RTX3090 上，eager + float16 仍然会产生 NaN；真正解决问题的是切换到 bfloat16 |
| **Attention 单独已具备判别力** | A1/A2/A3/A4 不再接近随机；A2 达到 0.7733 Acc / 0.8285 AUROC |
| **Score-only 强于 Output-only** | A2 (1536 维) 明显优于 A4 (40 维)，当前环境下 score 特征比 output 特征更强 |
| **Head selection 不再退化为随机** | 选中 head 的 val AUROC 落在 0.6069 - 0.6776，最强 head 为 L16-H05 (0.6776) |
| **Top-head 融合最有效** | A6 以 1792 维获得 0.8867 Acc，比 A0s 提升 +2.00% |
| **Output 更像排序补充信号** | A7 的 AUROC 高于 A0s（0.9254 vs 0.9184），但准确率下降；A8 加入 output 后拿到最高 AUROC |
| **Gated fusion 无净收益** | A9 只有 3 个样本发生路由变化，最终准确率回到 A0s 水平，没有额外纠错 |

### 2.3 NaN 复测结果

#### 2.3.1 最小诊断结论

在当前 Linux + RTX3090 环境中，最小 eager 诊断显示：

| 设置 | 检查项 | NaN 情况 |
|------|------|------|
| eager + float16 | 单样本 hidden[18] | 10752 / 10752 = 100% NaN |
| eager + float16 | 单样本 attn[17] | 588 / 588 = 100% NaN |
| eager + bfloat16 | 原始 hidden / attention 前向 | 0 NaN |
| eager + bfloat16 | hidden / score / output 三类提取器 | 0 NaN |

#### 2.3.2 新缓存 NaN 统计

| 特征类型 | 总元素数 | NaN 数 | NaN 比例 |
|------|:---:|:---:|:---:|
| Attention Scores (900 x 1536) | 1,382,400 | 0 | 0.0% |
| Attention Outputs (900 x 40) | 36,000 | 0 | 0.0% |

> 结论很直接：Linux + RTX3090 并没有自动修复 float16 的 eager NaN；bfloat16 才是这次复跑真正生效的解决方案。

## 四、结论

1. **NaN 问题已解决，但不是因为 Linux 本身**：eager + float16 在 Linux + RTX3090 上仍会产生 NaN；切换到 bfloat16 后，attention score / output 缓存的 NaN 比例降为 0%。
2. **全量 hidden baseline 提升到 0.8082**：这次 Phase 4 全量 hidden-only 结果高于历史 Phase 3 基线（0.7987）。
3. **Attention 特征不再是随机噪声**：在 NaN 消失后，A1/A2/A3/A4 全部明显高于随机，其中 A2 达到 0.7733 Acc / 0.8285 AUROC。
4. **最强准确率来自 top-head score 融合**：A6 达到 0.8867 Acc / 0.8865 F1，相对 A0s 提升 +2.00% Acc。
5. **最强 AUROC 来自三路融合**：A8 达到 0.9403 AUROC，说明 output 特征更适合作为排序补充信号而不是单独决策器。
6. **Gated Fusion 在本次运行中没有净收益**：虽然有 3 个样本发生融合路由变化，但没有带来净纠错。

## 五、当前推荐配置与运行方式

### 5.1 推荐配置

- 模型前向：eager attention
- 数值精度：bfloat16
- 运行环境：Linux + Ampere 及以上 GPU（RTX 3090 可用）

### 5.2 推荐命令

```bash
source "$(conda info --base)/etc/profile.d/conda.sh"
conda activate llm_hallucination
source ./.venv/bin/activate

# 推荐：按本次复跑配置运行完整 Phase 4
PHASE4_DTYPE=bfloat16 python -s scripts/run_phase4_full.py
```

### 5.3 兼容性说明

- main.py phase4 当前仍默认读取 src/config.py 中的 primary_dtype=float16，如果直接走这条路径，仍可能复现 eager NaN。
- 如需让主入口也走当前稳定配置，可将 src/config.py 中的 primary_dtype 改为 bfloat16。

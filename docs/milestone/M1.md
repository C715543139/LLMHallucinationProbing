## 4. 实践要点

### 4.1 路径管理统一使用 `pathlib.Path`

在 Windows 环境下，路径兼容性是基础稳定性的关键。Phase 1 中采用统一的路径配置与 `Path` 对象管理，避免硬编码字符串路径带来的问题。

### 4.2 数据划分强调可复现与防泄漏

Phase 1 的预处理输出不仅仅是“能切分数据”，还要满足后续实验要求：

- 训练 / 验证 / 测试划分比例清晰
- 三个集合之间无重叠样本
- 每个集合尽量保持真 / 假样本共存
- 训练集覆盖尽可能多的领域

### 4.3 模型侧优先验证最小闭环

在进入方法实验前，先验证最小可运行链路：

1. 模型可加载
2. tokenizer 可用
3. 单句前向传播成功
4. `hidden_states` 数量与模型层数一致
5. 最后 token 的隐藏状态可被稳定取出

这一步为后续 SAPLMA 与层分析实验奠定了接口基础。

### 4.4 测试先行保障 Phase 2 扩展

Phase 1 测试并不只是验收当前功能，也为 Phase 2 奠定了接口契约。例如：

- `TrueFalseDataset` 的字段结构
- 预处理 `.pt` 文件的加载方式
- 模型前向传播输出 hidden states 的约定

这些约定直接被后续隐藏状态特征提取与分类实验复用。
